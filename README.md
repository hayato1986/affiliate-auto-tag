# アフィリエイトID自動付与 Chrome拡張機能

Amazon.co.jp を閲覧する際に、設定済みのアフィリエイト（アソシエイト）IDをURLの `tag` パラメータに自動付与するChrome拡張機能です。

## 機能

- Amazon.co.jp の全ページに `tag` パラメータを自動付与
- グローバルON/OFF切替
- サイト別ON/OFF切替
- ダークモード対応（システム設定に連動）
- 設定はChromeアカウント間で同期

## インストール（開発版）

### 前提条件

- Node.js 18以上
- npm

### 手順

```bash
# 依存関係のインストール
npm install

# ビルド
npm run build
```

1. Chrome で `chrome://extensions` を開く
2. 右上の「デベロッパーモード」をONにする
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. `.output/chrome-mv3` フォルダを選択

### 開発モード

```bash
npm run dev
```

ホットリロード付きの開発サーバーが起動します。コード変更が即座に反映されます。

## 使い方

### 初期設定

1. インストール後、自動的に設定ページが開きます
2. Amazon.co.jp のトグルをONにする
3. アソシエイトIDを入力する（例: `mytag-22`）
4. 入力欄の下に「✓ 有効な形式です」と表示されれば設定完了
5. 設定は自動保存されます

### 動作確認

1. Amazon.co.jp の任意のページを開く
2. URLに `tag=あなたのID` が自動付与されていることを確認
3. ツールバーのアイコンをクリックすると現在の状態を確認できます

### ポップアップの見方

- **✓（緑）**: タグ付与中
- **!（オレンジ）**: ID未設定
- **トグル**: 全サイトの有効/無効を切替

### 無効にする

- ポップアップのトグルでグローバルON/OFFを切替
- 設定ページでサイト別にON/OFFを切替

## 技術仕様

### 無限ループ防止

`chrome.declarativeNetRequest` の2ルール構成でリダイレクトループを防止しています:

1. **allowルール（優先度2）**: `tag=` が既にあるURLにマッチ → リダイレクトをスキップ
2. **redirectルール（優先度1）**: `amazon.co.jp` にマッチ → `tag` パラメータを付与

allowルールが高優先度のため、タグ付与済みURLでは再リダイレクトが発生しません。

### 既存タグの扱い

他のアフィリエイトIDが既にURLに含まれている場合、上書きしません。これはAmazonアソシエイト規約上安全な挙動です。

### セキュリティ

- `innerHTML` 不使用（XSS対策）
- アフィリエイトID入力のバリデーション（インジェクション防止）
- `chrome.storage.sync` からの読み込み時に再バリデーション（改ざん対策）
- CSPで `script-src 'self'` のみ許可
- 権限は必要最小限（`declarativeNetRequest`, `storage`, `activeTab`）

### プライバシー

- 閲覧履歴は収集しません
- 外部サーバーへの通信は一切行いません
- 設定データはChromeの同期ストレージにのみ保存されます

### Chrome Web Store 審査のポイント

- **権限の正当性**: `declarativeNetRequest`（URLリダイレクト）、`storage`（設定保存）、`activeTab`（現在タブの状態表示）— いずれも機能に必須
- **host_permissions**: `*://*.amazon.co.jp/*` — 対象サイトに限定
- **プライバシーポリシー**: 外部通信なし・データ収集なしを明記する必要あり
- **Single purpose policy**: 「アフィリエイトIDの自動付与」という単一目的に合致

### バージョンアップ時

1. `wxt.config.ts` の `version` を更新
2. `npm run zip` で新しいZIPを生成
3. デベロッパーダッシュボードから新バージョンをアップロード
